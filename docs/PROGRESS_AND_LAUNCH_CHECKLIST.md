## 四·续二、2026-07-01 全站安全审查（第 4 轮，`jingcheng-saas` commit `4429a47`）

结论：**未发现资金主链路漏洞**（现金券核销/积分兑换三层防护、SQL 全参数化、鉴权体系均复核通过）；发现并已修复 1 个高危 + 若干中低危问题：

| 级别 | 问题 | 修复 |
|:----:|------|------|
| 🔴 高危 | 会员权益领取接口（CRMEB 支付回调）此前只信客户端传参 + 内部密钥可能仍是默认值 | 改为必须核对 CRMEB **真实已支付订单**（uid 匹配），按实付金额倒推档位；内部密钥缺失/默认值时启动打印安全警告 |
| 🟡 中危 | SN 拍照识别 4 个接口只校验登录、未校验"是否店员" | 补齐 `is_staff` 校验 |
| 🟡 中危 | 后台登录 `/admin/login` 无限流，可无限次试密码 | 加内存滑动窗口限流（连续错 5 次锁 15 分钟） |
| 🟡 中危 | 改密码接口存在换行注入风险（可借密码字段篡改 `.env` 其它变量） | 校验新密码禁止换行符 |
| 🟡 中危 | 管理后台商品详情预览 `v-html` 未净化 | 改用 `DOMPurify.sanitize` |
| 🟡 中危 | `xlsx` 依赖 npm 高危漏洞（原型污染 + ReDoS，官方未修复） | 切换到 SheetJS 官方 CDN 补丁版 `0.20.3` |
| 🔵 低危 | 核销码签名比较未做长度前置校验 | 补长度校验，避免极端伪造输入触发未捕获异常 |
| 🔵 加固 | 缺基础安全响应头 / CI 权限默认过宽 | 加 `X-Content-Type-Options`/`X-Frame-Options`/`Referrer-Policy`；CI 收紧为 `permissions: contents: read` |

验证：`npm test` 36/36 通过、`fzlsaas-admin` build 成功、`npm audit` 高危项清零、生产部署成功、安全响应头生产实测生效。

**残留非紧急项**（暂不处理，供后续参考）：
- CORS 仍为默认反射来源（未设 `CORS_ORIGINS` 白名单），当前靠 Cookie `SameSite=Strict` 兜底
- `esbuild`/`vite` 中危漏洞仅影响本地开发服务器，不影响生产构建产物，需升级 Vite 大版本（breaking change），可另排期
- `shunwei-membership` 仓库与 `jingcheng-saas/shunwei-api` 内各有一份已失效的旧 `.github/workflows/deploy.yml`（引用路径已不存在，实际不会被触发），非安全问题，可作后续代码清理项
