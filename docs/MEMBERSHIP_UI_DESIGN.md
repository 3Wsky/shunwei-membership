## 三、MVP1 页面规范

### 3.1 会员中心改造（`pages/annex/vip_paid`）

**改造点**（在现有 CRMEB 付费会员页基础上）：

```
┌─────────────────────────────────────┐
│ [头像] 昵称                          │
│ 🏅 299会员 · 2027-06-19 到期         │  ← 新增等级徽章组件
│ 累计节省 ¥xxx                        │
├─────────────────────────────────────┤
│ 会员权益图标行（沿用现有）            │
├─────────────────────────────────────┤
│ ┌─────────┐  ┌─────────┐            │
│ │ 199会员 │  │ 299会员 │  ← 横向卡片│
│ │ ¥199/年 │  │ ¥299/年 │            │
│ │赠199000 │  │赠299000 │  ← 积分说明 │
│ └─────────┘  └─────────┘            │
│ [立即开通 / 续费]                     │
├─────────────────────────────────────┤
│ 积分快捷入口 → 我的积分               │  ← 新增
│ 积分商城入口 → 兑换好礼               │  ← 新增
└─────────────────────────────────────┘
```

**交互**：
- 未登录：引导登录后展示
- 已开通：主按钮变「续费」，展示到期倒计时（<30 天橙色提醒）
- 开卡成功弹窗：展示赠送积分数量 + 「去积分商城看看」

### 3.2 我的积分页增强（`pages/users/user_integral`）

在现有头部总积分下方增加 **批次提示**（MVP1 后端就绪后接入）：

```
可用积分    12,450
├ 赠送积分（最早 2026-08-01 到期）  8,000
└ 充值积分（永久有效）              4,450
```

- 消耗顺序 FIFO，不在 UI 强调算法，仅展示「即将到期」提醒（7 天内橙色标签）。
- 底部增加「积分充值」入口（MVP3 启用，MVP1 可隐藏或置灰「即将开放」）。

### 3.3 积分商城（`pages/points_mall`）

| 场景 | UI 表现 |
|------|---------|
| 正常可兑换 | 仅显示积分价，**不显示库存数字** |
| 库存为 0 | 商品图半透明蒙层 + 按钮置灰「暂时无法兑换，过两天试试」 |
| 积分不足 | 按钮文案「积分不足」+ 引导「去赚积分」 |
| 兑换成功 | 跳转订单详情，配送方式固定「到店自提」 |

**商品卡片**：
- 积分价使用 `--brand-primary`，字号 36rpx 加粗
- 建议定价区间展示：5,000–20,000 积分（运营配置，设计不做硬编码）

### 3.4 积分兑换订单（自提核销）

- 订单状态流：`待核销` → `已核销`
- 核销码以大号等宽字体展示，支持一键复制
- 店员核销端 MVP1 走 CRMEB 现有 `order_verific`，UI 不改

### 3.5 超管免审开关（后台）

- shunwei-api `/admin` 增加开关 Toggle
- 状态文案：「开启：店员可直接核销积分兑换 / 关闭：需三级审批（MVP2）」
- MVP1 默认 **开启**

### 10.1 API → UI 数据绑定

#### GET `/api/membership/me`

| 响应字段 | UI 展示位置 | 展示规则 |
|---------|------------|---------|
| `tierCode` | 会员徽章 | `tier_199` → 铜金徽章；`tier_299` → 紫金徽章；`none` → 隐藏徽章 |
| `tierName` | 徽章旁文案 | 「199 会员」/「299 会员」 |
| `expiresAt` | 到期日行 | 格式 `YYYY-MM-DD 到期` |
| `daysRemaining` | 到期提醒 | `<30` 天时橙色 `#FF9500` 标签「即将到期」 |
| `giftIntegralPending` | 开卡后提示条 | `>0` 时显示「您有 xxx 积分待领取」+ 按钮调 claim-gift |
| `memberShipId` | 档位卡片选中态 | 6=199 卡高亮，7=299 卡高亮 |

#### GET `/api/integral/summary`

| 响应字段 | UI 展示位置 | 展示规则 |
|---------|------------|---------|
| `totalIntegral` | 积分页头部大数字 | 千分位格式化 `12,450` |
| `giftRemaining` | 批次行 | 「赠送积分」+ 子文案 |
| `giftNearestExpireAt` | 赠送积分子文案 | `最早 YYYY-MM-DD 到期`；7 天内橙色 |
| `rechargeRemaining` | 批次行 | 「充值积分（永久有效）」 |
| `expiringSoonAmount` | 顶部横幅（可选） | `>0` 时黄色提示条 |

#### POST `/api/membership/claim-gift`

| 场景 | UI |
|------|-----|
| 成功 | 全屏半透明蒙层 + 礼花动效（可简化为弹窗） |
| 弹窗标题 | 「开通成功」 |
| 弹窗正文 | 「已赠送 **199,000** 积分」 |
| 主按钮 | 「去积分商城」→ `/pages/points_mall/index` |
| 次按钮 | 「知道了」关闭 |
| 幂等重复调用 | Toast「积分已到账」不重复弹窗 |
