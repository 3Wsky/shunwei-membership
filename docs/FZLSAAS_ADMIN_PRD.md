## 二、各模块详细需求

### 2.1 数据看板

**现有**：会员总数、今日核销、待结算、待审批（占位 `--`，未接 API）

**新增指标**：

| 指标 | 口径 | 刷新 |
|------|------|------|
| 今日积分新增 | `sw_integral_ledger` 当日 `type IN (grant,gift,recharge,legacy_import)` 之和 | 5min |
| 今日积分消耗 | 当日 `type IN (exchange,expire,deduct)` 之和 | 5min |
| 今日新注册会员 | `eb_user` 当日新增且绑定微信 | 5min |
| 今日审批通过数 | `sw_approval_request` 当日终态=approved | 5min |

**优化**：
- 第二行：近 7 日积分新增/消耗折线图（ECharts）
- 快捷入口：待审批 N 条 → 点击跳审批页并带筛选
- 时间切换：今日 / 近7日 / 近30日

**API**：`GET /api/admin/dashboard/summary?range=today|7d|30d`

### 2.2 会员管理（重点）

#### 2.2.1 会员分类维度

| 分类标签 | 判定规则 | 筛选用途 |
|----------|----------|----------|
| 普通用户 | 无有效 `sw_user_membership` | 潜客运营 |
| 199 会员 | `tier=199` 且未过期 | 权益运营 |
| 299 会员 | `tier=299` 且未过期 | 高价值客户 |
| 店员 | `eb_user.is_staff=1` | 权限管理 |
| 店长 | `sw_store_manager` 表有记录 | 审批初审 |
| 商家 | `sw_merchant.login_uid` 存在 | 异业管理 |

支持**多标签叠加**（如「299 会员 + 店员」）。

#### 2.2.2 列表字段

UID、昵称、手机号、会员等级、到期日、积分余额（可用/冻结）、现金券余额、归属店员、注册时间、最近消费审批

#### 2.2.3 单会员操作（抽屉/详情页）

- 查看积分批次明细、现金券批次、审批历史
- **开通/撤销店员**（绑定 `division_id` 门店）
- **开通/撤销商家**（跳转商家表单预填 UID）
- **手动开通会员**（199/299 + 赠积分，走现有 `POST /api/admin/membership/grant`）
- **发放积分**（指定数量 + 批次类型：赠送/充值 + 有效期）
- **发放现金券**（指定金额 + 备注）
- **变更归属店员**（修改 `spread_uid` 或 `sw_member_attribution`）

#### 2.2.4 批量操作

- 勾选多条 → 批量发积分 / 批量发现金券 / 批量开通会员
- CSV 导入：`uid,action,amount,tier,remark`（上限 500 行/次，异步任务 + 结果下载）
- 操作前二次确认 + 写 `sw_admin_audit_log`

**API 缺口**：
- `GET /api/admin/members/list`（分页+多维筛选）
- `GET /api/admin/members/:uid/detail`
- `PUT /api/admin/members/:uid/staff-role`
- `POST /api/admin/members/batch-grant`
- `POST /api/admin/integral/grant`（单发/批量）

### 2.3 店员管理

#### 2.3.1 列表

| 字段 | 说明 |
|------|------|
| 店员 UID/姓名/手机 | 来自 `eb_user` |
| 所属门店 | `division_id` → 门店名称 |
| 是否店长 | `sw_store_manager` |
| 发展会员数 | 归属该店员的 `sw_member_attribution` 或 `spread_uid` 计数 |
| 待审批/已通过 | 该店员发起的审批统计 |
| 名片状态 | 是否已配置头像/简介 |

#### 2.3.2 详情

- **发展会员列表**：可导出 CSV
- **审批记录**：该店员作为 `staff_uid` 的全部 `sw_approval_request`
- **名片配置**（后台编辑 → 小程序展示）：
  - 头像、姓名、职位、个人简介
  - 门店名称、地址、电话、营业时间
  - 微信二维码（可选）

#### 2.3.3 小程序店员名片页（新增）

路径建议：`pages/staff/card`（分享带 `staffUid` 参数）

- 展示店员信息 + 门店地址（地图跳转）
- 底部「联系店员」→ 拨打电话 / 加微信
- 新用户通过名片注册/登录后 **自动绑定归属店员**

**与 CRMEB 分销关系**：复用 `eb_user.spread_uid` 作为归属字段；后台「店员管理」即原分销上级的运营视图，**小程序侧隐藏佣金/提现**，只保留归属统计。

**API 缺口**：
- `GET /api/admin/staff/list`
- `GET /api/admin/staff/:uid/stats`
- `GET/PUT /api/admin/staff/:uid/card`
- `GET /api/staff/:uid/card`（小程序公开读）

### 2.4 商家管理（增强）

**现有**：仅「创建商家」表单

**增强为完整 CRUD**：

| 区块 | 字段 |
|------|------|
| 基础信息 | 名称、类目、联系人、电话、绑定 UID |
| 门店资料 | 地址（省市区+详细）、经纬度、门头图（多图）、营业时间 |
| 权限 | 核销开关、结算周期备注 |
| 核销记录 | 分页列表：时间、客户 UID、核销金额、操作人、结算状态 |
| 结算台账 | 待结算总额、已结算记录、标记已结算（线下结算，系统只记账） |

**API 缺口**：
- `GET /api/admin/merchant/list`
- `GET/PUT /api/admin/merchant/:id`
- `GET /api/admin/merchant/:id/verify-logs`
- `GET /api/admin/merchant/settlement-summary`

### 2.5 审批管理（增强）

**现有**：超管待办列表 + 终审（刚修复 Cookie 鉴权）

**增强**：

| 能力 | 说明 |
|------|------|
| 全量记录 | 全部状态：pending_store / pending_admin / approved / rejected |
| 筛选 | 店员、门店、金额区间、档位、日期、小票号 |
| 审批详情 | 消费金额、档位演算、小票图、各级审批人+时间+意见 |
| 一键免审 | 开关：`sw_system_config.approval_auto_pass`（已有 integral-mall 免审，扩展至消费审批） |
| 免审规则 | 可配置：仅积分商城 / 含消费审批 / 指定门店 |
| 店长初审 | **仅在小程序**（已有 approval/center）；PC 只看记录+终审 |
| 驳回重提 | 展示驳回原因链；支持超管撤销终批（24h 内，写审计） |

**审批流（确认）**：
```
店员提交 → 店长小程序初审 → 超管 PC 终审 → 自动开通权益
         ↘ 免审开启时：店员提交 → 自动通过（仍留记录）
```

**API 缺口**：
- `GET /api/admin/approval/list`（全量，非仅 todos）
- `GET /api/admin/approval/:id`
- `PUT /api/admin/config/approval-auto-pass`
- `POST /api/admin/approval/:id/revoke`

### 2.6 商品管理

维持「纯展示」定位，**不做购物车/下单**。

- 列表：封面、名称、价格、展示开关、排序
- 操作：上下架、编辑、价签一键导入（已有）
- 优化：批量上下架、拖拽排序

**现状**：✅ 后端+前端基本满足，P2 抛光。

## 三、权限模型

| 角色 | 后台权限 |
|------|----------|
| **超级管理员** | 全部菜单 + 终审 + 批量发放 + 系统设置 |
| **店长**（可选二期） | 只读会员/店员本店 + 审批记录；无发券/开商家 |
| **运营**（可选） | 商品/积分商城/抽奖内容管理 |

MVP：**仅超管**登录 FZLSaas；店长审批走小程序。

## 五、体验优化清单

1. **统一操作反馈**：批量任务显示进度条 + 失败行号
2. **会员搜索**：UID / 手机号 / 昵称模糊搜
3. **危险操作**：撤销店员/商家/会员 需输入确认文字
4. **空状态引导**：无会员时链到「批量导入说明」
5. **面包屑上下文**：从店员详情跳会员详情保留返回栈
6. **移动端适配**（低优）：超管偶尔手机看数据，看板响应式即可
